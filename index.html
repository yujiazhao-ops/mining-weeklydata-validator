周报 · 算力指标（巡查表同步）
M1
管理机器数（台）
SUMIFS(H列持有机器数, 日期=周末, BN列=分类)。总揽不过滤采购分类（含客户机器/电力矿场），U本位/币本位列额外过滤BJ列。取周末最后一天快照。
M2
管理算力（P）
SUMIFS(K列理论总算力, 日期=周末, BN列=分类)，逻辑同M1。理论总算力为机器全量算力，不扣除故障台数。
M3
NS持有算力（P）
SUMIFS(BA列ns算力, 日期=周末, BN=分类, BJ∈[U本位/币本位])，取周末快照。不含客户机器/电力矿场。
M4
NS有效算力（P）
SUMIFS(BM列ns实际算力, 日期∈周内, BN=分类, BJ∈[U本位/币本位], BK≠需剔除场地) ÷ 7。周内7天日平均，BK列「需剔除场地」的行不计入。
M5
周算力达成率
NS有效算力 ÷ (NS持有算力 + 超频附加值)。超频附加值按分类×本位填写，达成率 ≥90% 绿，≥70% 橙，低于70% 红。
M6
剔除异常场地达成率
NS有效算力 ÷ (SUMIFS(BL列ns持有去故障, 日期=周末, BK=正常) + 超频附加值)。分母只累加BK列为「正常」的行。
M7
我方代托 达成率（特殊）
不使用NS列，改用：周内日均I列(24h平均算力/1000) ÷ 周内日均X列(架上理论算力/1000)。客户机器无NS数据，故用巡查表实测算力代替。
周报 · 收益指标（收益表同步）
R1
产出BTC（枚）
SUMIFS(AC列预估BTC收入, 日期∈周内, 矿工号BN=分类, 矿工号BJ=本位)。分类和本位通过矿工号从巡查表同步BN/BJ列映射，不依赖收益表AT/AS列（可能存在分类缺失，如AMIDN子账户）。
R2
USD收入（$）
SUMIFS(AF列最近24h BTC价值USD, 日期∈周内, 分类, 本位)，周内累计。
R3
托管运维费（$）
= USD收入 − 毛利。收益表中无独立成本列，成本已隐含在毛利公式中，反推得出。
R4
毛利（$）
SUMIFS(AH列理论毛利, 日期∈周内, 分类, 本位)，周内累计。各利润模式的毛利计算逻辑见每日巡查校验工具R13规则。
R5
电费占比
= 托管运维费 ÷ USD收入，反映电费及运维成本在总收入中的比重。
场地明细 · 计算规则
S1
场地分类来源
从「自营算力托管场地明细表」的「SADG观察者链接」Sheet读取。按类别和场地名称分组，每个场地含一个或多个子账户（矿工号）。矿场管理类子账户与其他分类重叠，数据同时计入多个分类进行加总。
S2
托管数量 / 总算力
按场地所有子账户，SUMIFS(H列持有机器数 / K列理论总算力, 日期=周末)。取周期最后一天快照，总算力(T) = 理论总算力(P) × 1000，不扣故障台数。
S3
周平均算力有效率
= 周内日均24h算力(I列) ÷ 期间最后一天理论总算力(K列，不扣故障)。日均 = 7天24h算力之和 ÷ 7，含零值天（停电/维护等），零值天会标注提示但不排除。分母取周末最后一天快照值。
S4
场地周毛利
按场地所有子账户，SUMIFS(AH列理论毛利, 日期∈周内)，周内累计，数据来自收益表同步。
数据说明 & 注意事项
分类（BN列）：SAD / 新合资 / 老合资 / 其他项 / 我方代 / NS / 已经出
本位（BJ列）：U本位 / 币本位 / 客户机器 / 电力矿场；后两者只计入总揽不拆分本位
日期读取：xlsx.js 以 UTC 时间解析日期，工具统一用 getUTCDate() 转换，避免时区偏移导致日期错位一天
重名列处理：巡查表含多个重名列（如「ns持有算力」），工具用 header:1 模式按列索引读取，保留首次出现的列
超频附加算力：按分类×本位填写，总揽自动加总，自动保存到浏览器本地
对比周：从同一文件中按对比日期范围重新计算；数量类环比用百分比，达成率类用 pp（百分点）
零值天标注：场地有效率若某天24h算力为0（停电/维护），标注「含X天零值」，数值仍含0值计算
AMIDN子账户收益：收益表AT列分类可能缺失，工具通过巡查表BN列映射确保正确归类，比直接SUMIFS收益表更准确
