算力指标（巡查表同步）
M1
管理机器数（台）
SUMIFS(H列持有机器数, 日期=周末, BN列=分类)，总揽不过滤采购分类，U本位/币本位列额外过滤BJ列
M2
管理算力（P）
SUMIFS(K列理论总算力, 日期=周末, BN列=分类)，同上
M3
NS持有算力（P）
SUMIFS(BA列ns算力, 日期=周末, BN列=分类)，取周末最后一天快照
M4
NS有效算力（P）
SUMIFS(BM列ns实际算力, 日期∈周内, BN=分类, BJ∈[U本位/币本位], BK≠需剔除场地) ÷ 7，取周内7天平均
M5
周算力达成率
NS有效算力 ÷ (NS持有算力 + 超频附加值)，超频附加值按分类单独维护
M6
剔除异常场地达成率
NS有效算力 ÷ (SUMIFS(BL列ns持有去故障, 日期=周末, BK=正常) + 超频附加值)
收益指标（收益表同步）
R1
产出BTC（枚）
SUMIFS(AC列预估BTC收入, 日期∈周内, 矿工号分类=BN分类, 采购分类=本位)，按矿工号从巡查表映射分类
R2
USD收入（$）
SUMIFS(AF列btc价值USD, 日期∈周内, 分类, 本位)
R3
托管运维费（$）
USD收入 − 毛利（表格没有直接的成本列，成本已计入毛利计算）
R4
毛利（$）
SUMIFS(AH列理论毛利, 日期∈周内, 分类, 本位)
R5
电费占比
托管运维费 ÷ USD收入
数据说明
分类（BN列）：SAD / 新合资 / 老合资 / 其他项 / 我方代 / NS / 已经出
本位（BJ列）：U本位 / 币本位 / 客户机器 / 电力矿场，后两者只计入总揽不拆分
超频附加算力：人工调整值，加在NS持有算力上作为达成率分母，按分类单独维护，自动保存到本地
对比周：从同一份文件中按对比日期范围重新计算，可选择任意两周对比
环比变化：(本周 − 对比周) ÷ |对比周|，绿色为正向，红色为负向
